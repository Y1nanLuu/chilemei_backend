# “吃了没”大学生校园美食年度报告生成软件开发项目计划书



# 一、项目概述



## 1.1 项目背景与定位



针对大学生校园饮食场景，打造一款**轻量化、个性化、社交化**的美食记录与推荐软件。软件聚焦大学生个人饮食数据月度、年度分布，兼顾校园食堂+周边校外美食，实现“随手记美食、自动生报告、智能推美食”的核心价值，满足大学生记录饮食、复盘口味偏好、发现周边热门美食、避雷难吃食物的需求。



## 1.2 核心目标



- 搭建便捷的美食录入体系，支持大学生自定义记录食堂、校外各类美食信息

- 数据库记录用户饮食记录数据，生成专属年度美食报告，可视化展示口味偏好等

- 基于用户饮食数据和平台大数据，实现个性化美食推荐、相似口味推送

- 打造简洁易用的界面，适配大学生使用习惯，支持报告分享、数据留存等附加功能



## 1.3 适用人群



热爱美食、乐意实时记录与分享的在校大学生或教职工。

# 二、需求详细说明



## 2.1 核心业务需求



### 2.1.1 美食记录功能



- **自定义美食录入**：用户可新建美食条目，必填/选填信息包括：美食名称、就餐地点（食堂楼层/窗口、校外店铺名称+地址）、价格、二维评价（赞/踩）、五维评分（夯/顶级/人上人/NPC/拉完了）、打卡日期、文字评价、实拍图片上传。

- **快速记录入口**：简化录入流程，支持历史美食一键重复记录，降低用户操作成本

- **数据管理**：支持历史记录修改、删除、查看，按喜欢/避雷等标签筛选



### 2.1.2 每日推荐功能



- **每日推荐卡片**：每天系统推荐“开发者精选”或者“昨日热门”美食卡片，卡片中心照片展示，下方文字说明，包括美食名称、就餐地点、价格。右下角三个小图标，分别为“赞”、“踩”、“想吃”，实时动态展示用户评价。卡片下方设置评论区，用户可以评论发表评价，增强互动。



### 2.1.3 美食榜单功能



- **日榜/周榜/总榜**：根据用户每天上传的美食与评价，统计每日/每周/至今所有美食的点赞、点踩情况，根据赞数、踩数排序，展示美食推荐/避雷榜单。用户可选“看所有人”和“只看自己”功能。



### 2.1.4 年度报告生成功能





- **年度饮食总览**：全年打卡次数、总消费金额、就餐地点分布（食堂VS校外）

- **口味偏好复盘**：最爱吃的菜品TOP5、最常打卡的窗口/店铺

- **消费行为分析**：月度消费趋势、人均单次消费、平价/轻奢美食占比

- **特色趣味标签**：生成个性化称号（如“食堂干饭王”“校外探店达人”“减脂干饭人”等）

- **报告导出分享**：支持生成图片版、PDF版年度报告，可分享至社交平台



## 2.2 非功能需求



- **界面体验**：年轻化、简约清新的UI设计，操作流程简单易懂，无复杂功能

- **性能要求**：录入、查询、报告生成响应速度快，支持离线查看历史记录

- **数据安全**：用户饮食数据本地存储+云端备份，保障个人隐私不泄露

- **兼容性**：优先适配手机端（微信小程序/APP），支持主流安卓、iOS系统

## 4.2 技术选型



- **前端技术栈**：采用**Taro框架**搭建微信小程序，搭配**Vue3**语法开发，选用**NutUI组件库**快速构建页面交互；该方案支持跨端适配，后续可根据需求灵活调整框架、组件库或转其他端开发。

- **前后端通信方案**：通过**HTTP/HTTPS接口**实现前后端数据交互，前端发起请求调用后端接口，传递美食录入、用户信息等参数；后端接收请求、处理数据后返回JSON格式响应结果，配套接口文档统一对接规范，保障数据传输稳定。

- **后端基础架构**：选用轻量后端语言（Python/Java/Node.js均可）搭建服务，搭建RESTful API接口体系，处理业务逻辑、数据校验、权限控制等核心操作，暂不锁定具体语言，可根据开发熟练度择优选用。

- **数据库选型**：采用**MySQL关系型数据库**存储核心数据，规划用户表、美食记录表、评价表、榜单统计表、日志表等基础表结构，实现数据持久化存储与高效查询；后续可根据数据量扩容、优化索引提升读写效率。

- **推荐算法**：**待规划**，暂不指定具体算法方案，后续结合用户饮食数据量、推荐场景需求，再确定标签匹配、协同过滤或简易规则引擎等实现方式。

- **数据可视化**：沿用ECharts图表库，适配Taro框架渲染年度报告各类可视化图表，展示消费趋势、口味偏好、打卡数据等内容。

# 五、人员分工与职责



由于开发人数（3人）限制，需要一人承担多个分工职责。



- **产品经理**：牵头全周期需求管控，第1周完成需求细化与原型绘制；全程跟进迭代进度，把控产品体验与功能逻辑。

- **UI设计师**：第1周完成首页推荐、记录、榜单、我的四大核心界面UI设计；第2周补全全量次级界面、弹窗、图标视觉设计；负责年度报告视觉风格定制、界面排版优化；同步输出切图、设计规范，配合前端开发还原视觉效果，全程跟进UI细节迭代。

- **前端开发（Taro+Vue3+NutUI）**：基于Taro框架搭建微信小程序工程，适配Vue3语法开发；第2周落地注册登录页面开发，后续迭代完成美食录入、榜单、推荐、个人中心、年度报告等全页面开发；实现NutUI组件调用、页面交互逻辑、数据渲染与接口联调；负责ECharts可视化图表适配、离线缓存功能开发、页面性能优化与兼容性调试。

- **后端开发**：第3周完成MySQL数据库表结构设计与搭建；搭建轻量后端服务，开发RESTful API接口，实现用户注册登录、美食上传、数据隔离、榜单统计、评论互动、年度报告生成等核心业务逻辑；负责数据校验、权限控制、接口文档编写；预留推荐算法开发端口，后期对接个性化推荐逻辑，保障数据读写效率与安全性。

- **测试工程师**：跟随敏捷迭代节奏，开展周度模块测试、全量功能联调测试；聚焦微信小程序兼容性、接口稳定性、数据准确性、用户体验测试；跟踪BUG上报、验证修复结果，出具测试报告；负责压力测试、离线功能测试、数据安全测试，把控版本上线质量，配合开发闭环问题。
